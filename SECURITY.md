# Security Policy

## Supported releases

Security fixes are evaluated for the latest public release and the active preview line of each runnable repository. Documentation-only repositories do not imply a production support commitment.

## Reporting a vulnerability

Please use GitHub private vulnerability reporting on the affected repository when available. Do not open a public issue containing credentials, customer data, exploit details or unredacted logs.

Include the affected repository and version, reproduction steps, impact, environment and a minimal sanitized proof. The project will acknowledge receipt, assess scope and coordinate disclosure. No response-time SLA is implied unless a separate written agreement exists.

## Scope boundary

Full Spectrum projects are early open-source governance tools. A passing test suite or local pilot label is not a production security certification, regulatory approval or warranty.
