# Support

Use GitHub Issues for reproducible bugs, documentation defects and feature proposals. Use Discussions when enabled for design questions and general adoption guidance.

Before requesting help, include the repository, exact version or commit, operating system, command, sanitized logs and expected/actual result. Never post tokens, credentials, customer data or private configuration.

Public repositories are provided without an implied production SLA. Enterprise deployment, compliance and operational responsibility remain subject to a separate written agreement.
